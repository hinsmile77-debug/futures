## 2. 채용 항목 상세

### 2-1. HFT 타임스탬프 동기화 ★★★★★

**배경**:
```
문제 시나리오:
  09:32:00.000 — 키움 서버 1분봉 확정
  09:32:00.150 — 미륵이 수신
  09:32:00.380 — 신호 생성
  09:32:00.620 — 주문 접수 (620ms 지연)

결과:
  백테스트: 388.50 진입 → 수익 0.5pt
  실전:     388.55 진입 → 수익 0.0pt (사라짐)
```

**현재 시스템 한계**:
- Circuit Breaker: API 5초 이상 지연만 감지
- 300ms 내의 미세 지연 감지 불가
- 백테스트-실전 괴리의 가장 흔한 원인

**구현**:
```python
# collection/kiwoom/latency_sync.py
T_api   = kiwoom_recv_time()
T_local = time.perf_counter()
latency = (T_local - T_api) * 1000  # ms

if latency > 300:
    slippage_weight *= 1.5   # 슬리피지 가중치 상향
    if latency > 1000:
        signal = "HOLD"      # 1초 이상 → 진입 차단
```

**모듈**: `collection/kiwoom/latency_sync.py`
**반영**: Phase 1 Week 1 (키움 API 연결과 동시 구현)

### 2-2. Hurst Exponent ★★★★★ (MDD 킬러)

**핵심 가치**:
```
H > 0.55: 추세 지속 → 추세추종 우위
H = 0.50: 랜덤 워크 → 어느 전략도 무용
H < 0.45: 평균 회귀 → 횡보 (잦은 손절 구간)

실증:
  횡보 구간 진입 차단만으로 MDD -25~40%
  Renaissance, Citadel 등 사용 표준 지표
```

**v6.5 미시 레짐과의 시너지**:
```
ADX > 25 AND H > 0.55 → 강한 추세 (신뢰도 +15%)
ADX > 25 AND H < 0.45 → 가짜 추세 (진입 차단)
ADX < 20 AND H < 0.45 → 명확 횡보 (역추세만)
```

**Gemini 코드 오류 수정**:
```python
# Gemini 원본 (오류)
hurst_idx = reg[0] * 2.0

# 수정본 — Variance 분석에서만 ×2, R/S 분석은 기울기 그대로
hurst_idx = reg[0]   # polyfit 기울기 = H (R/S 분석 기준)
```

**1분봉 적정 파라미터**:
```
max_lag = 20~30  (반응성 최적)
임계값: H < 0.45 → 횡보 (진입 금지)
        H > 0.55 → 추세 (진입 허용)
        0.45 ≤ H ≤ 0.55 → 데드존 (신중 진입)
```

**모듈**: `features/technical/hurst_exponent.py`
**반영**: Phase 2 Week 5 (Circuit Breaker와 함께 — 안전장치 단계)

### 2-3. 적응형 켈리 공식 ★★★★★

**기존 vs 신규**:
```
기존 (정적 켈리):
  신뢰도 기반 고정 배수
  → 시스템 성적 무관하게 동일 사이즈

신규 (적응형 켈리):
  f* = (p × (b+1) - 1) / b
  p = 최근 20회 실전 승률
  b = 최근 20회 손익비

상황별 자동 조정:
  승률 65%, 손익비 1.5 → f* = 0.42 (적극적)
  승률 50%, 손익비 1.0 → f* = 0.00 (진입 중단)
  승률 40%, 손익비 0.8 → f* < 0   (사실상 정지)
```

**안전장치**: 하프 켈리 유지 (f* / 2), 최대 배율 1.5 캡핑

**모듈**: `strategy/entry/adaptive_kelly.py`
**반영**: Phase 2 Week 6 (슬리피지 시뮬레이터와 함께)

### 2-4. VPIN ★★★★

**학술 배경**:
- Easley, Lopez de Prado, O'Hara (2012)
- 2010 Flash Crash 유일 사전 감지 지표
- 시카고대 부스 스쿨 표준 교재

```
VPIN = |매수 거래량 - 매도 거래량| / 총 거래량
       (time이 아닌 volume bucket 기준 정규화)

해석:
  VPIN < 0.3: 정상 거래
  VPIN > 0.5: 정보 거래자 활동 의심
  VPIN > 0.7: 강한 정보 거래자 → 큰 움직임 임박
```

**자동 진입 조건 강화**:
```python
def auto_entry_condition(signal, vpin_percentile):
    if (signal.confidence >= 0.70 and
        signal.foreign_aligned and
        vpin_percentile >= 0.90):
        return "AUTO_EXECUTE"    # 3중 조건
    return "MANUAL_CONFIRM"
```

**모듈**: `features/supply_demand/vpin.py`
**반영**: Phase 3 Week 8

### 2-6. 호가 취소 속도 (Cancel-to-Fill) ★★★★

**스푸핑 감지 원리**:
```
스푸핑 패턴:
  1. 매수 호가 대량 표시 (상승 유도)
  2. 가격 상승 → 본인 매도 체결
  3. 매수 호가 즉시 취소

감지:
  cancel_ratio = cancel_count / fill_count
  > 3.0 → 스푸핑 의심 → 반대 방향 가중치
```

**기존 OFI와의 차이**:
```
기존 OFI·LOBID: 정적 (특정 시점 잔량 측정)
신규 Cancel Ratio: 동적 (변화 속도 추적)
→ 보완 관계, 중복 아님
```

**모듈**: `features/supply_demand/cancel_ratio.py`
**반영**: Phase 3 Week 8
