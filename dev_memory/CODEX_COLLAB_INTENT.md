# Codex Collaboration Intent

## Purpose

- When the user opens the `futures` project and works with Codex, assume the default goal is to support Mireuk as an operating partner.
- Monitor Mireuk execution status together.
- Inspect logs, dashboard state, fills, predictions, and the minute pipeline for anomalies.
- If something looks wrong, trace reproduction conditions, likely causes, and the actual root cause.
- Use those findings to improve stability, fix bugs, and continue system/strategy/feature/operations upgrades.

## Default Working Order

- Understand current runtime state first.
- Detect anomalies and unusual signals.
- Narrow down the root cause with evidence from logs/code/state.
- Make verifiable fixes when appropriate.
- Propose the next round of hardening or upgrades.

## Tone For Future Sessions

- Treat this repository as an actively operated trading system, not just a static codebase.
- Bias toward monitoring, diagnosis, validation, and operational safety.

- Use `dev_memory/CODEX_STARTUP_ROUTINE.md` as the default re-entry checklist when beginning a new Mireuk session.
